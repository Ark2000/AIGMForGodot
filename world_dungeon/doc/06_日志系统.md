## 第六层：日志系统

> 目标：从event_log中产生可读的、有视角的叙事日志。

### 6.1 设计原则

日志系统是纯输出层，不影响任何游戏状态。它订阅事件，将事件翻译成可读文本。

**两种运行模式**：
```
FOLLOW模式：跟随某个特定实体（通常是冒险者）
  只输出该实体能"感知到"的事件
  感知范围 = 当前房间 + Graph距离1以内的房间

WORLD模式：输出所有重要事件
  过滤掉低重要度事件（普通怪物的日常游荡）
  保留：战斗、死亡、升级、重大发现、冒险者事件
```

### 6.2 事件重要度分级

每种事件有默认的重要度，用于WORLD模式过滤：

```
CRITICAL  : yendor_retrieved, boss_died, adventurer_died
HIGH      : entity_died, entity_leveled_up, item_equipped
MEDIUM    : entity_attacked, skill_used, item_picked_up
LOW       : entity_moved, modifier_applied
TRACE     : action_timer_ready（调试用，通常不输出）
```

### 6.3 日志渲染器（logger/renderer.lua）

每种事件对应一个渲染函数，输出自然语言字符串。

**渲染函数的输入**：
- event数据
- world引用（用于查询实体名字、位置等上下文）
- perspective（当前视角配置）

**渲染示例**：

```
entity_attacked事件 →
  命中：  "[T=342] Aldric 挥剑砍中了地精，造成 12 点伤害。(地精 HP: 15→3)"
  暴击：  "[T=342] Aldric 找准时机，重击地精！造成 24 点暴击伤害。(地精 HP: 15→0)"
  未命中："[T=342] Aldric 的攻击被地精灵巧地躲开了。"

entity_died事件 →
  怪物死亡："[T=343] 地精倒下了。掉落了 3 枚铜币。"
  冒险者死亡："[T=500] Aldric 在第3层倒下了。享年 level 4，击杀 23 只怪物。"

entity_leveled_up事件 →
  "[T=800] Aldric 升至 5 级！获得新技能：盾击。"

yendor_retrieved事件 →
  "[T=9823] Aldric 手持Yendor护符，踏出了地牢的入口。
            *** 进入 Hall of Fame ***
            用时 9823 tick，等级 12，击杀 187 只怪物。"
```

### 6.4 perspective.lua

**Perspective配置**：
```lua
-- 跟随某个冒险者
{
    mode         = "follow",
    target_eid   = eid,
    perception_range = 1,        -- Graph距离
    min_importance = "LOW",
}

-- 世界视角
{
    mode         = "world",
    min_importance = "HIGH",
    focus_floor  = nil,          -- nil=全部楼层，number=只看某层
}
```

**FOLLOW模式的过滤逻辑**：
```
对每个事件：
  找出事件涉及的room_id
  如果 room_id 在 target的感知范围内 → 输出
  否则 → 跳过

特例：
  target自身相关的事件（target是attacker或defender）→ 永远输出
  CRITICAL级别事件 → 永远输出（无论距离）
```

### 6.5 日志输出配置

在`data/config/simulation.lua`中：
```lua
return {
    max_ticks = 10000,
    
    log = {
        enabled      = true,
        mode         = "follow",    -- "follow" | "world"
        
        -- follow模式：自动跟随第一个生成的冒险者
        -- 冒险者死亡后切换到下一个
        auto_follow  = true,
        
        -- 输出目标
        output       = "stdout",    -- "stdout" | "file"
        file_path    = "logs/sim.log",
        
        -- 每N tick输出一次世界状态摘要（即使follow模式）
        summary_interval = 1000,
    },
    
    max_adventurers = 3,
    adventurer_spawn_interval = 500,
}
```

**世界状态摘要（每1000tick）**：
```
=== Tick 1000 摘要 ===
活跃冒险者：2人（Aldric lv4 第2层, Brynn lv2 第1层）
各层种群：第1层 12只, 第2层 8只, 第3层 15只
本周期击杀：47
Hall of Fame：0人
```

**验收标准**：
- FOLLOW模式只输出感知范围内的事件
- 每种主要事件类型都有对应的文本模板
- 冒险者死亡后自动切换跟随目标
- 每1000tick输出世界摘要
- 跑10000tick，日志量在可读范围内（不爆炸）
