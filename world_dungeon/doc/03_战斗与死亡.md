## 第三层：战斗与死亡

> 目标：实体之间能够发生战斗，有伤害、死亡、loot掉落。

### 3.1 阵营系统

**faction组件**：
```
faction = {
    id       : string
    hostility : {
        [faction_id] : "aggressive" | "neutral" | "friendly"
    }
}
```

**FactionSystem工具函数**（不是独立system，是被其他system调用的工具模块）：
- `get_hostility(world, eid_a, eid_b)` → 返回a对b的态度
- `is_enemy(world, eid_a, eid_b)` → 互为敌对时返回true

stats = {

### 3.3 战斗触发

**在ai_system中**：当实体ready时，检查当前房间是否有敌对实体。

```
优先级：
1. 如果同房间有敌人 → intent = attack（优先于移动）
2. 如果感知范围内（Graph距离N）有敌人 → intent = move（向敌人方向移动）
3. 否则 → 执行当前行为原型的默认逻辑（游荡等）
```

### 3.4 战斗系统（combat_system）

**Priority：35**

**执行条件**：实体有`action_timer`(ready=true) + `ai_intent`(action="attack")

**伤害结算流程**：

数值公式定义在`data/config/combat_formulas.lua`：
```lua
return {
    -- 基础物理伤害
    physical_damage = function(attacker_stats, defender_stats)
        local base = attacker_stats.attack - math.floor(defender_stats.defense * 0.5)
        return math.max(1, base)
    end,
    
    -- 命中判定
    hit_chance = function(attacker_stats, defender_stats)
        local base = 0.75
        local speed_diff = attacker_stats.speed - defender_stats.speed
        return math.min(0.95, math.max(0.05, base + speed_diff * 0.03))
    end,
    
    -- 暴击判定
    crit_chance = function(attacker_stats)
        return 0.05  -- 基础5%，后续可被技能/装备修改
    end,
    
    crit_multiplier = 2.0,
}
```

**结算步骤**：
1. 取攻击方和防御方的有效属性（经过modifier计算，见第四层）
2. 命中判定：随机数 < hit_chance → 命中，否则miss
3. 暴击判定：命中后再判断暴击
4. 计算伤害值
5. 对防御方hp扣减
6. emit `entity_attacked` 事件（包含完整结算数据）
7. 如果hp <= 0，emit `entity_died` 事件
8. 重置攻击方action_timer为`attack_cooldown`

**entity_attacked事件数据**：
```
{
    attacker_id  : eid
    defender_id  : eid
    hit          : bool
    crit         : bool
    damage       : number
    hp_before    : number
    hp_after     : number
    room_id      : room_id
}
```

### 3.5 死亡处理（spawn_system订阅entity_died）

**entity_died事件触发以下流程**：

1. **Loot掉落**：读取死亡实体的`loot_table`组件，按概率生成物品实体，设置location为当前房间
2. **经验分配**：找同房间的敌对存活实体，分配经验（emit `xp_gained`事件）
3. **实体销毁**：`world:destroy_entity(eid)`
4. **生态记录**：更新种群计数（ecology_system订阅此事件）

**loot_table组件**：
```
loot_table = {
    { item_id="copper_coin", count={min=1,max=5}, chance=0.8 },
    { item_id="rusty_dagger",                    chance=0.1 },
    { item_id="health_potion",                   chance=0.15 },
}
```

**xp_gained事件数据**：
```
{
    receiver_id : eid
    amount      : number
    source_id   : eid   -- 击杀了谁
}
```

### 3.6 物品实体基础结构

物品是Entity，此阶段只实现基础部分：

```
item_info = {
    def_id    : string   -- 引用item_def
    name      : string
    tags      : string[]
    weight    : number
    value     : number
    stackable : bool
    stack_count : number  -- stackable时有效
}

location = {
    type     : "ground" | "inventory" | "equipped"
    room_id  : eid | nil     -- type=ground时有效
    owner_id : eid | nil     -- type=inventory/equipped时有效
    slot     : string | nil  -- type=equipped时有效
}
```

**验收标准**：
- 同房间的敌对实体会互相攻击
- 攻击有命中/miss/暴击判定
- 死亡实体正确掉落物品
- 物品实体存在于地面上
- event_log中有完整的战斗记录
