## 第二层：空间与地牢

> 目标：生成一个可遍历的地牢Graph，实体能在其中移动。

### 2.1 地牢数据模型

**地牢结构**：多层，每层是一个独立的无向图。

**Room作为Entity**：
每个房间是一个普通ECS实体，拥有以下组件：

```
room_info = {
    type    : string   -- "entrance","corridor","chamber","lair","treasure","stairs","boss"
    floor   : number   -- 第几层
    template: string   -- 引用room_template定义
}

connections = {
    -- 邻接表，key是任意标识，value是连接描述
    [conn_id] = {
        target_room : eid
        state       : "open" | "locked" | "hidden" | "blocked"
        key_id      : string | nil  -- 锁需要哪把钥匙
        one_way     : bool          -- 是否单向（比如落差）
    }
}

room_state = {
    light    : "bright" | "dim" | "dark"
    tags     : string[]   -- 动态标签，比如"cleared","cursed","visited"
}
```

注意：Room没有position组件（position是给移动实体用的），Room本身就是空间的节点。

**Floor（层）的数据**：
Floor不是实体，只是一个结构记录，存在World上：
```
world.dungeon = {
    floors = {
        [1] = {
            rooms       : { eid → true }  -- 这层所有房间
            entrance    : eid              -- 入口房间
            exit        : eid              -- 下楼梯房间
            boss_room   : eid | nil
        },
        ...
    },
    total_floors : number
}
```

### 2.2 地牢生成器（dungeon/generator.lua）

**输入**：`floor_config`数据定义
**输出**：完整的房间实体网络，写入`world.dungeon`

**floor_config数据格式**：
```lua
-- data/floor_configs/floor_1.lua
return {
    id    = "floor_1",
    floor = 1,
    
    room_count    = { min=8,  max=14 },
    room_types    = {
        corridor  = 0.4,
        chamber   = 0.3,
        lair      = 0.2,
        treasure  = 0.1,
    },
    
    guaranteed = { "entrance", "stairs_down", "boss_room" },
    
    connectivity = "sparse",  -- "sparse"|"normal"|"dense" 影响冗余连接数量
    
    -- 怪物生成参数（生成器负责初始填充）
    monster_density = 0.6,    -- 每个eligible房间的填充概率
    monster_pool    = { "goblin", "goblin_archer", "giant_rat" },
    
    -- 物品生成
    loot_density    = 0.3,
    
    difficulty      = 1,      -- 影响怪物等级偏移
}
```

**生成算法**（简单可靠优先）：
1. 按`room_count`随机确定数量
2. 按`room_types`权重生成对应类型的房间实体
3. 确保`guaranteed`中的房间类型存在
4. 用随机生成树（spanning tree）连接所有房间，保证连通性
5. 按`connectivity`参数追加额外连接（增加冗余路径）
6. 按`monster_density`和`monster_pool`在eligible房间中生成初始怪物
7. 按`loot_density`在eligible房间中生成初始物品

### 2.3 topology.lua（运行时Graph查询）

提供对地牢Graph的查询接口，System通过这个模块做空间查询，不直接操作raw数据。

**核心接口**：
- `get_neighbors(world, room_id)` → 返回所有可通行的相邻房间id列表（过滤locked/hidden/blocked）
- `get_connections(world, room_id)` → 返回所有连接（包括不可通行的），供AI判断门是否可以开
- `find_path(world, from_room, to_room)` → BFS最短路径，返回room_id数组，不可达返回nil
- `find_entities_in_range(world, room_id, range, filter_fn)` → Graph距离N以内的所有实体，filter_fn可选
- `get_floor_of_room(world, room_id)` → 返回该房间在第几层
- `get_stairs_down(world, room_id)` → 从当前层找下楼梯的房间

### 2.4 行动计时器系统（action_timer_system）

**Priority：5**（最先执行）

**相关组件**：
```
action_timer = {
    cooldown_max : number   -- 当前行动结束后重置到这个值（由上一次行动决定）
    cooldown_cur : number   -- 当前剩余冷却，每tick-1
    ready        : bool     -- cooldown_cur <= 0 时为true
}
```

**每tick执行**：
遍历所有有`action_timer`的实体，`cooldown_cur - 1`，如果归零则设`ready = true`。

### 2.5 移动系统（movement_system）

**Priority：30**

**执行条件**：实体有`position` + `action_timer`(ready=true) + `ai_intent`(action="move")

**执行流程**：
1. 查询所有ready且intent为move的实体
2. 检查目标房间是否与当前房间相邻且连接可通行
3. 如果合法：`world:move_entity_to_room(eid, target)`，emit `entity_moved`事件，重置action_timer（cooldown_max由`move_cooldown`属性决定）
4. 如果不合法：清空intent，不消耗行动（AI下次重新决策）

**move_cooldown**：来自实体的`actor`组件，数据定义中指定。

### 2.6 AI系统（ai_system）

**Priority：10**（在movement之前，负责产生intent）

此阶段只实现最基础的AI，复杂行为后续迭代。

**执行条件**：实体有`action_timer`(ready=true) + `ai_behavior` + `position`，且没有`ai_intent`（避免覆盖未执行的intent）

**此阶段只实现一个行为原型**：`random_wander`
- 从当前房间的可通行邻居中随机选一个
- 设置`ai_intent = { action="move", target=room_id }`

action_timer ready=true
    ↓
ai_system 产生 ai_intent

```
ai_intent = {
    action : string   -- "move"|"attack"|"use_skill"|"pick_up"|"wait"
    target : any      -- 依action不同而不同
                      -- move    → room_id
                      -- attack  → eid
                      -- use_skill → { skill_id, target_eid }
                      -- pick_up → item_eid
                      -- wait    → nil
}
```

**验收标准**：
- 生成一个完整的地牢Graph，所有房间连通
- 生成若干实体（怪物），每个实体有position和action_timer
- 跑1000 tick，实体在地图上随机游荡
- event_log中能看到entity_moved事件
- 没有实体移动到不相邻的房间
